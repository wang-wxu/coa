I/O系统

I/O Communication

计算机I/O通信

In a Computer System, in addition to the CPU and Main Memory, Input/Output (I/O) is a major functional subsystem. Input/Output System provides a mechanism for communication between the CPU and the external world. I/O subsystem connects the external devices like Keyboard, Mouse, Monitor, Joystick, Pendrive and internal devices like Hard Disk, CD to the Computer.

在计算机系统中，除了中央处理器和主存储器之外，输入/输出(I/O)是一个主要的功能子系统。输入输出系统为CPU和外部设备之间的通信提供了一种机制。I/O子系统将键盘、鼠标、显示器、操纵杆、驱动器等外部设备和硬盘、CD等内部设备连接到计算机上。

We know the functioning principles and characteristics of these devices are widely different from each other. So, what can simplify the technical issues involved in connecting these wide varieties of devices? The answer is to establish applicable communication standard(s).

我们知道这些设备的工作原理和特性彼此之间有很大的不同。那么，怎样才能简化连接这些种类繁多的设备所涉及的技术问题呢?答案是建立适用的通信标准。

Hard I/O

硬件 I/O

Hard I/O refers to data transfer between a computer and a physical device, like a keyboard or mouse.

硬件 I/O 是指计算机和物理设备(如键盘或鼠标)之间的数据传输。

Soft I/O

软件 I/O

Soft I/O refers to input and output that occurs over a network, for example, when a computer communicates with other computers, streams media or transfers files.

软件 I/O 是指通过网络进行的输入和输出，例如，当一台计算机与其他计算机通信、流媒体或传输文件时。

Intra System and Inter System Communication

系统内和系统间通讯

Whether it is Intra or Inter System Communication, a medium is required to carry the communication and also a protocol or standard to carry out the communication successfully. The most familiar one is the Internet as a Wired or Wireless communication. It has protocol standards defined by IEEE. Similarly, every device has a communication protocol which is called the Interface Standard.

无论是系统内通信还是系统间通信，都需要一种媒介来进行通信，也需要一种协议或标准来成功地进行通信。最熟悉的是作为有线或无线通信的互联网。它具有IEEE定义的协议标准。类似地，每个设备都有一个称为接口标准的通信协议。

Computer Bus

计算机总线

Alternatively known as an address bus, data bus, control bus, or local bus, a bus is a link between components or devices connected to a computer. For example, a bus carries data between a CPU and the system memory via the motherboard.

总线是连接到计算机的组件或设备之间的链路，也称为地址总线、数据总线、控制总线或本地总线。例如，总线通过主板在CPU和系统内存之间传输数据。

A bus is either a parallel or serial bus, and either an internal bus (local bus) or an external bus (expansion bus).

总线可以是并行总线或串行总线，也可以是内部总线(本地总线)或外部总线(扩展总线)。

A computer or device's bus speed is measured in MHz, e.g., an FSB may operate at a frequency of 100 MHz. The throughput of a bus is measured in bits per second or megabytes per second.

计算机或设备的总线速度以兆赫为单位测量，例如，FSB可以在100兆赫的频率下工作。总线的吞吐量以比特/秒或兆字节/秒来衡量。

Examples of computer buses

Today, many of the buses listed above are no longer used or are not as common. Below is a listing of the most common buses and how they are used with a computer.

今天，上面列出的许多总线不再使用或不那么常见。下面列出了最常见的总线以及它们如何与计算机一起使用。

eSATA and SATA - Computer hard drives and disc drives.

eSATA 和 SATA - 计算机硬盘驱动器和磁盘驱动器。

PCIe - Computer expansion cards and video cards.

PCIe - 计算机扩展卡和视频卡。

USB - Computer peripherals.

USB - 计算机外围设备。

Thunderbolt - Peripherals connected through a USB-C cable.

Thunderbolt - 通过USB-C电缆连接的外设。

System Bus

系统总线

The medium and the protocol together is called a BUS. A Bus is a shared communication link consisting of a set of connecting cables/wires which carry signals that are required as per the Interface Standard's definition. A bus facilitates to achieve thed "Where- What -How" of the Communication amongst the various subsystems. We will focus on Internal communication carried out by System bus. A system Bus has three components Address, Data and Control Signals.

媒体和协议一起被称为总线。总线是一种共享的通信链路，由一组连接电缆/电线组成，这些电缆/电线根据接口标准的定义携带所需的信号。总线有助于实现各个子系统之间“何处-什么-如何”的通信。我们将重点讨论通过系统总线进行的内部通信。系统总线由地址、数据和控制信号三部分组成。

I/O Controller

I/O控制器

I/O Controller will typically have Data Register(s), Status Register(s), Control Register(s), Address decoding logic and Control Circuitry. The I/O Controller is connected to the system bus. Whenever the I/O controller wants to use the bus, it has to contend and obtain. All communication from the CPU and Memory happens via these registers shown in the diagram. These registers are given a unique address for each I/O controller.

I/O控制器通常有数据寄存器、状态寄存器、控制寄存器、地址解码逻辑和控制电路。I/O控制器连接到系统总线上。每当I/O控制器想要使用总线时，它都必须进行争用和获取。所有来自CPU和内存的通信都通过图中所示的这些寄存器进行。这些寄存器为每个I/O控制器提供一个唯一的地址。

I/O Control Methods

I/O 控制方式

In order for a system to communicate with external devices and other computers, it uses controllers to transfer data between the various components of a computer. Within a computersystem, a controller is a device or program that manages or directs the operations of other devices or programs. Typical computer systems contain both hardware- and software-based controllers that serve as intermediaries between I/O devices. There are four control method types.

为了使系统与外部设备和其他计算机通信，计算机使用控制器在计算机的各个组件之间传输数据。在计算机系统中，控制器是管理或指导其他设备或程序的操作的设备或程序。典型的计算机系统包含基于硬件和基于软件的控制器，它们作为I/O设备之间的中介。有四种控制方式类型。

Programmed I/O

程序控制 I/O

通过编程控制 I/O，处理器对每一次I/O数据传输执行一条程序指令。这种控制方法是最慢的，但最简单的程序。

Interrupt-Driven I/O

中断驱动控制 I/O

A processor is interrupted when an I/O device needs to transfer data. The processor will halt its normal processing to execute the device's instructions, then resume normal processing afterward. In this case, a processor, also known as CPU (central processing unit), interprets and executes instructions it receives from a computer program. Processors are physical devices within a computer system that perform calculations and process input data. Processors act as the brains of the computer and supply the computing power needed for a system to manage the constant stream of instructions it receives. This control method is faster than programmed I/O, but also is more difficult to program.

当I/O设备需要传输数据时，处理器中断。处理器将停止其正常处理以执行设备的指令，然后恢复正常处理。在这种情况下，处理器，也称为CPU(中央处理单元)，解释并执行它从计算机程序接收到的指令。处理器是计算机系统中执行计算和处理输入数据的物理设备。处理器充当计算机的大脑，为系统提供处理接收到的连续指令流所需的计算能力。这种控制方法比编程I/O更快，但也更难编程。

Direct Memory Access (DMA)

直接存储器存取(DMA)

This occurs when the processor only has to initiate the data transfer and the DMA controller takes over to complete it. This control method is very fast compared to programmed and interrupt-driven I/O, and also more efficient because it requires minimal processor involvement.

当处理器只需要启动数据传输，DMA控制器接管完成传输时，就会发生这种情况。与编程和中断驱动的I/O相比，这种控制方法非常快，而且效率更高，因为它只需要最少的处理器参与。

Advanced Programmable Interrupt Controller (APIC)

高级可编程中断控制器(APIC)

In this method, when the processor receives notification that a device is ready to transfer data, it sends a signal to the APIC. The APIC then sends an interrupt to the processor so it can execute the program. This control method is the fastest of the four, but is most complex in terms of programming.

在这种方法中，当处理器接收到设备准备好传输数据的通知时，它向APIC发送一个信号。然后APIC发送一个中断给处理器，这样它就可以执行程序了。这种控制方法是四种方法中最快的，但在编程方面是最复杂的。

Input Devices

输入设备

Input device enables the user to send data, information, or control signals to a computer. The Central Processing Unit (CPU) of a computer receives the input and processes it to produce the output.

输入设备使用户能够向计算机发送数据、信息或控制信号。计算机的中央处理单元(CPU)接收输入并对其进行处理以产生输出。

Output Devices

输出设备

The output device displays the result of the processing of raw data that is entered in the computer through an input device. There are a number of output devices that display output in different ways such as text, images, hard copies, and audio or video.

输出设备显示通过输入设备输入计算机的原始数据的处理结果。有许多输出设备以不同的方式显示输出，如文本、图像、硬拷贝、音频或视频。
